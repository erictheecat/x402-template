# DNS Route Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a paid `POST /v1/dns` endpoint that returns requested DNS records plus best-effort TLS certificate metadata with a standard x402 receipt.

**Architecture:** Keep the Fastify route thin and move DNS/TLS logic into a dedicated helper module. Reuse the existing `/v1` payment, idempotency, and receipt envelope patterns while introducing a fixed route-specific `$0.003` price constant that is shared across route, catalog, and payment middleware.

**Tech Stack:** Fastify, TypeScript, Node built-ins (`node:dns/promises`, `node:tls`), Vitest
